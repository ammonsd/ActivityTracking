Project Overview

About the System

The Task Activity Management System is a comprehensive enterprise application designed to streamline time tracking, expense reporting, and project management workflows. Built with modern Java technologies and industry-standard frameworks, this system provides a robust, scalable solution for organizations managing both time-based deliverables and expense reimbursements.

Core Functionality:

  • Task Activity Tracking: Log time spent on client projects with detailed phase tracking (Development, Testing, etc.), weekly timesheet views, and comprehensive filtering and reporting capabilities
  • Expense Management: Submit business expenses with receipt attachments, track approval workflows through multiple status stages (Draft ‚Üí Submitted ‚Üí Approved ‚Üí Reimbursed), and maintain detailed audit trails
  • Role-Based Access Control: Database-driven authorization system with flexible role and permission management. Four pre-configured user roles (USER, ADMIN, EXPENSE_ADMIN, GUEST) with granular permissions stored in the database, allowing administrators to create custom roles and modify permissions without code changes
  • Approval Workflows: Multi-step approval processes for expenses with email notifications, approval notes, and rejection handling with detailed feedback
  • Analytics & Reporting: Interactive dashboards with charts for time analysis, client distribution, project trends, and team performance metrics (admin only)
  • Data Export: CSV export functionality for tasks, expenses, and timesheets enabling integration with external accounting and reporting systems
  • Data Import: Bulk CSV import functionality for TaskActivity and Expense records with comprehensive validation, duplicate detection, multiple date format support (YYYY-MM-DD, MM/DD/YYYY, DD-MMM-YYYY), and automated PowerShell script for streamlined data migration
  • Security Features: Account lockout after failed login attempts, password expiration policies (90 days), forced password updates, session timeout management (30 minutes), JWT token revocation and blacklisting, and magic number validation for file uploads

Technical Architecture:

  • Backend: Java 21 (Latest LTS) with Spring Boot 3 framework, leveraging Spring MVC, Spring Data JPA, and Spring Security
  • Frontend: Dual interface approach - Angular 19 (standalone components) for the modern dashboard and reports, Thymeleaf templates for traditional server-rendered pages (task/expense lists, forms)
  • Database: PostgreSQL with Hibernate ORM, HikariCP connection pooling, and Spring Data repositories for data persistence
  • Build & Deploy: Maven build system with Docker containerization, AWS ECS deployment, Jenkins CI/CD pipeline running on WSL Ubuntu, and CloudFormation infrastructure as code
  • API Documentation: Springdoc OpenAPI with Swagger UI for REST API exploration and testing
  • File Storage: AWS S3 integration for receipt attachments with lifecycle policies and secure presigned URLs

Key Differentiators

This system demonstrates real-world enterprise patterns including email notifications (AWS SES), file upload handling with S3 storage, comprehensive audit logging, role-based permissions with approval workflows, and a hybrid frontend architecture combining Angular SPA features with traditional server-side rendering for optimal performance and user experience.

‚úÖ 12-Factor App Compliance

This application fully adheres to the 12-Factor App methodology, demonstrating enterprise-grade cloud-native architecture and modern DevOps best practices.

  • I. Codebase: Single Git repository, multiple environment deployments
  • II. Dependencies: Explicitly declared in Maven (pom.xml) and npm (package.json)
  • III. Config: Environment variables for all configuration, AWS Secrets Manager integration
  • IV. Backing Services: PostgreSQL as attached resource, swappable via configuration
  • V. Build, Release, Run: Strict separation via Maven, Docker, and CI/CD pipeline
  • VI. Processes: Stateless application design, shared database backend
  • VII. Port Binding: Self-contained HTTP service with embedded Tomcat
  • VIII. Concurrency: Horizontal scaling via process replication (ECS, Kubernetes)
  • IX. Disposability: Fast startup, graceful shutdown, health check endpoints
  • X. Dev/Prod Parity: Identical Docker images across all environments
  • XI. Logs: Stdout/stderr event streams captured by CloudWatch
  • XII. Admin Processes: One-off tasks in identical environment (DatabaseInitializer, scripts)
